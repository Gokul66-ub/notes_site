Chapter 01

What is HTML?

HTML (HyperText Markup Language) is the standard language used to build the skeleton of every webpage. It tells the browser what content to display and how it is organised — not how it looks (that's CSS) or how it behaves (that's JavaScript).

🏠

The House Analogy

Think of building a website like building a house:
  • HTML = The walls, floors, rooms — the structure
  • CSS = The paint, furniture, decorations — the style
  • JavaScript = The electricity, plumbing, appliances — the logic

HTML is not a programming language. It has no loops, conditions, or calculations. It is a markup language — you "mark up" content with tags to give it meaning.
💡 Fun Fact: HTML was invented by Tim Berners-Lee in 1991. The World Wide Web started in 1989. The current version is HTML5 (2014, still updated).
🔖

Tags, Elements & Attributes — The Building Blocks

Tag = An instruction wrapped in angle brackets: <tagname>
Element = Opening tag + content + closing tag: <p>Hello</p>
Attribute = Extra info inside the opening tag: <a href="url">
Self-closing tag = Tags with no content and no closing tag: <br>, <img>, <hr>
HTML CODE
<!-- An element: tag + content + closing tag -->
<p>This is a paragraph element.</p>

<!-- An element with an attribute -->
<a href="https://google.com">Click me</a>

<!-- Self-closing tag (no content, no closing tag) -->
<br>
<hr>
<img src="photo.jpg" alt="A photo">
LIVE OUTPUT
⚠️ Always write tags in lowercase. <body> ✅ not <BODY> ❌. Always close your tags — forgetting </p> or </div> causes layout bugs.
Chapter 02

The <head> Element & Meta Tags

The <head> contains metadata — information about your page that is not shown on the screen. This is where you set the title, character encoding, viewport, and link stylesheets.

🧠

Complete Head Section Explained

charset: Tells browser to use UTF-8 encoding so special characters (ñ, é, ₹) display correctly.
viewport: Makes the page responsive on mobile devices — without it, mobile zooms out.
title: Shown on browser tab and used by Google in search results.
description: Shown under the link in Google search results (SEO).
link rel="stylesheet": Connects an external CSS file to style the page.
link rel="icon": Sets the small icon on the browser tab (favicon).
COMPLETE <head> EXAMPLE
<!DOCTYPE html>
<html lang="en">

  <head>

    <!-- 1. Character Encoding: lets browser show all characters -->
    <meta charset="UTF-8">

    <!-- 2. Viewport: makes page responsive on phones/tablets -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">

    <!-- 3. Title: appears on browser tab + Google search -->
    <title>My Awesome Website</title>

    <!-- 4. Description: shown in Google search results (SEO) -->
    <meta name="description" content="A portfolio site for Jane Doe">

    <!-- 5. Author: who made this page -->
    <meta name="author" content="Jane Doe">

    <!-- 6. Favicon: small icon shown on browser tab -->
    <link rel="icon" type="image/x-icon" href="favicon.ico">

    <!-- 7. Link CSS: connect an external stylesheet -->
    <link rel="stylesheet" href="style.css">

    <!-- 8. Internal CSS: write styles directly inside the head -->
    <style>
      body { background-color: lightyellow; }
    </style>

  </head>

</html>
💡 Nothing inside <head> is shown on the page — it's invisible metadata. Only <title> appears on the tab. Everything the user sees goes inside <body>.
Chapter 03

Document Structure

Every valid HTML page must follow this exact skeleton. Think of it as the mandatory frame every house must have before you can add rooms.

🏗

The Complete HTML5 Skeleton

<!DOCTYPE html> — Always the very first line. Tells the browser this is an HTML5 document. Without it, browsers go into "quirks mode" and render things differently.

<html lang="en"> — The root element. Everything lives inside this. lang helps screen readers and search engines know the page language.

<head> — Invisible metadata section (title, charset, CSS links).

<body> — Everything the user sees goes here. Only one <body> per page.
HTML SKELETON
<!-- 1: DOCTYPE must be FIRST, before everything -->
<!DOCTYPE html>

<!-- 2: Root element wraps the entire document -->
<html lang="en">

  <!-- 3: HEAD — invisible info about the page -->
  <head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width, initial-scale=1.0">
    <title>My Page Title</title>
  </head>

  <!-- 4: BODY — everything the user sees -->
  <body>

    <h1>Hello, World!</h1>
    <p>Welcome to my first webpage.</p>

  </body>

</html>
LIVE OUTPUT
💡 Comments: Use <!-- your comment here --> anywhere to add notes. The browser ignores them — only you can see them in the source code. Great for explaining your code!
Chapter 04

Text Basics & Formatting

Headings give your page hierarchy. Paragraphs hold your text. Formatting tags add meaning like bold, italic, quotes, and more.

📰

Headings — h1 to h6

HTML has 6 heading levels. <h1> is the most important (main page title — use only once per page). <h6> is the least important. Search engines (Google) use headings to understand your page structure — so use them in order, like a book's chapters.
CODE
<h1>Heading 1 — Main Title</h1>
<h2>Heading 2 — Section</h2>
<h3>Heading 3 — Sub-section</h3>
<h4>Heading 4</h4>
<h5>Heading 5</h5>
<h6>Heading 6 — Smallest</h6>
OUTPUT
¶

Paragraphs, Line Breaks & Dividers

<p> creates a block of text with automatic spacing above and below.
<br> = line break (moves to next line without starting a new paragraph).
<hr> = horizontal rule — a visible divider line between sections.
<pre> = pre-formatted text — preserves spaces and line breaks exactly as typed.
CODE
<p>This is the first paragraph.</p>

<p>Line one.<br>Line two (same paragraph).</p>

<hr>

<pre>This    text
   preserves
      spacing!</pre>
OUTPUT
✏️

Text Formatting Tags

<b> = Bold (visual only, no extra meaning) | <strong> = Bold + important meaning (prefer this!)
<i> = Italic (visual only) | <em> = Italic + emphasis meaning (prefer this!)
<u> = Underline | <s> = Strikethrough | <mark> = Highlighted text
<sup> = Superscript (e.g. x²) | <sub> = Subscript (e.g. H₂O)
<q> = Inline quotation | <blockquote> = Block-level quote
<abbr title=""> = Abbreviation with tooltip on hover
CODE
<p><b>Bold</b> and <strong>Strong</strong></p>
<p><i>Italic</i> and <em>Emphasis</em></p>
<p><u>Underline</u> and <s>Strikethrough</s></p>
<p><mark>Highlighted text</mark></p>
<p>x<sup>2</sup> and H<sub>2</sub>O</p>
<p><q>Short inline quote</q></p>
<blockquote>A longer block quotation.</blockquote>
<p><abbr title="HyperText Markup Language">HTML</abbr> (hover me!)</p>
OUTPUT
Chapter 05

Links — The <a> Anchor Tag

Links connect pages together — they are what makes the "HyperText" in HTML. The anchor tag <a> can link to external pages, internal sections, files, email addresses, and phone numbers.

🔗

All Types of Links

href = "hypertext reference" — the destination URL.
target="_blank" = Opens link in a new tab (use for external links).
href="#id" = Jumps to an element with that id on the same page (anchor link).
mailto: = Opens the user's email app.
tel: = Opens phone dialer on mobile.
title = Tooltip text shown when hovering over the link.
CODE
<!-- External link: opens in same tab -->
<a href="https://www.google.com">Go to Google</a>

<!-- External link: opens in NEW tab -->
<a href="https://www.google.com"
   target="_blank">Google (New Tab)</a>

<!-- Internal link to another file -->
<a href="about.html">About Page</a>

<!-- Anchor link: jump to #section on same page -->
<a href="#contact">Jump to Contact</a>

<!-- Email link -->
<a href="mailto:[email protected]">Email Me</a>

<!-- Phone link (works on mobile) -->
<a href="tel:+911234567890">Call Me</a>

<!-- Link with tooltip -->
<a href="https://google.com"
   title="Visit Google Search">Hover for tooltip</a>

<!-- Clickable image (image inside a link) -->
<a href="https://google.com">
  <img src="logo.png" alt="Google" width="80">
</a>
OUTPUT
Chapter 06

Images — <img>

The <img> tag embeds images. It is self-closing (no ending tag). Always include alt text for accessibility and SEO.

🖼

Image Tag & All Attributes

src = path to image (local file or full URL).
alt = alternate description if image fails to load. Screen readers read this. Google uses it for image SEO. Always required.
width / height = size in pixels or %. Set only one to avoid distortion.
title = tooltip text when hovering.
Relative path = images/photo.jpg (file in your project folder).
Absolute URL = https://site.com/photo.jpg (file on the internet).
CODE
<!-- Local image (from your images/ folder) -->
<img src="images/photo.jpg"
     alt="A photo of me">

<!-- Image from the internet -->
<img
  src="https://picsum.photos/300/150"
  alt="Random sample image"
  width="300"
  title="Hover tooltip here">

<!-- If image fails, alt text shows instead -->
<img src="broken.jpg"
     alt="This shows when image breaks"
     width="200">
OUTPUT
⚠️ Never skip alt="". It's required for accessibility. Blind users rely on screen readers that read the alt text aloud. Also, search engines use alt to understand images.
Chapter 07

Lists — Ordered, Unordered & Nested

Lists organise related items. HTML has three kinds: unordered (bullets), ordered (numbers), and description lists.

📋

All List Types with Nesting

<ul> = Unordered list (bullets) — for items with no particular order.
<ol> = Ordered list (numbers/letters) — for steps or ranked items.
<li> = List item — goes inside both <ul> and <ol>.
<dl> <dt> <dd> = Description list (term + definition pairs).
Nested lists = Put a <ul> or <ol> inside an <li> to create sub-lists.
CODE
<!-- Unordered list (bullets) -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered list (numbers) -->
<ol>
  <li>Learn HTML</li>
  <li>Learn CSS</li>
</ol>

<!-- Ordered with letters -->
<ol type="A">
  <li>Option A</li>
  <li>Option B</li>
</ol>

<!-- Nested list -->
<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend
    <ul>
      <li>Python</li>
    </ul>
  </li>
</ul>

<!-- Description list -->
<dl>
  <dt>HTML</dt>
  <dd>Structure of a webpage</dd>
  <dt>CSS</dt>
  <dd>Style of a webpage</dd>
</dl>
OUTPUT
Chapter 08

Tables — Rows, Columns, Merge

Tables display data in rows and columns — like a spreadsheet. Use colspan to merge columns horizontally and rowspan to merge rows vertically.

📊

Basic Table Structure

<table> = Container for the whole table.
<thead> = Header section | <tbody> = Body section | <tfoot> = Footer section.
<tr> = Table row | <th> = Header cell (bold, centred) | <td> = Data cell.
<caption> = Optional title shown above the table.
colspan="n" = Cell spans n columns sideways.
rowspan="n" = Cell spans n rows downward.
CODE
<table border="1" style="width:100%;border-collapse:collapse">

  <caption>Student Results</caption>

  <thead>
    <tr>
      <th>Name</th>
      <!-- colspan: merges 2 columns -->
      <th colspan="2">Scores</th>
    </tr>
    <tr>
      <th></th><th>Maths</th><th>Science</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <!-- rowspan: this cell spans 2 rows -->
      <td rowspan="2">Arjun</td>
      <td>90</td><td>85</td>
    </tr>
    <tr>
      <!-- No first td here: Arjun spans into it -->
      <td>88</td><td>92</td>
    </tr>
    <tr>
      <td>Priya</td>
      <td>95</td><td>97</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <td>Total</td>
      <td colspan="2">3 Tests</td>
    </tr>
  </tfoot>

</table>
OUTPUT
Chapter 09

Forms & All Input Types

Forms collect user input — login, signup, contact, search. Every input should have a linked label for accessibility.

📝

Complete Form with Every Input Type

<form action="" method=""> = Container. action= where to send data. method="get" = data visible in URL. method="post" = hidden, secure (use for passwords).
<label for="id"> = Links text label to its input by matching id. Click label = focuses input.
required = Field cannot be empty. placeholder = Hint text. disabled = Grayed out, non-editable. value = Pre-filled value.
CODE
<form action="#" method="post">

  <!-- Text -->
  <label for="name">Name:</label><br>
  <input type="text" id="name" name="name"
         placeholder="Your full name" required>

  <!-- Password -->
  <label for="pwd">Password:</label><br>
  <input type="password" id="pwd" name="pwd"
         placeholder="Enter password" required>

  <!-- Email -->
  <label for="email">Email:</label><br>
  <input type="email" id="email" name="email">

  <!-- Number -->
  <label for="age">Age:</label><br>
  <input type="number" id="age" min="1" max="120">

  <!-- Date -->
  <label for="dob">Date of Birth:</label><br>
  <input type="date" id="dob" name="dob">

  <!-- Radio (single select — same name groups them) -->
  <p>Gender:</p>
  <input type="radio" id="m" name="gender" value="male">
  <label for="m">Male</label>
  <input type="radio" id="f" name="gender" value="female">
  <label for="f">Female</label>

  <!-- Checkbox (multiple select) -->
  <p>Skills:</p>
  <input type="checkbox" id="html" name="skills">
  <label for="html">HTML</label>
  <input type="checkbox" id="css" name="skills" checked>
  <label for="css">CSS (pre-checked)</label>

  <!-- Dropdown select -->
  <label for="city">City:</label><br>
  <select id="city" name="city">
    <option value="">-- Select --</option>
    <option value="chennai">Chennai</option>
    <option value="delhi" selected>Delhi</option>
    <option value="mumbai">Mumbai</option>
  </select>

  <!-- Textarea: multi-line text -->
  <label for="msg">Message:</label><br>
  <textarea id="msg" rows="3" cols="30"
            placeholder="Write here..."></textarea>

  <!-- Range slider -->
  <label>Rating:</label><br>
  <input type="range" min="1" max="10" value="7">

  <!-- Colour picker -->
  <label>Pick colour:</label>
  <input type="color" value="#00d4ff">

  <!-- File upload -->
  <label>Upload:</label>
  <input type="file">

  <!-- Buttons -->
  <br><input type="submit" value="Submit">
  <input type="reset" value="Reset">
  <button type="button">Plain Button</button>

</form>
OUTPUT
Chapter 10

Semantic HTML Tags

Semantic tags describe the purpose of content — not just its appearance. They help search engines understand your page structure and improve accessibility for screen readers.

🏛

Page Layout with Semantic Tags

<header> = Top section: site logo, nav, tagline.
<nav> = Navigation links (menu).
<main> = Main content of the page. Only one per page.
<section> = A thematic group of content with its own heading.
<article> = Self-contained content (blog post, news article).
<aside> = Side content (ads, related links, author bio).
<footer> = Bottom: copyright, social links, contact.
<figure> + <figcaption> = Image with a caption.
<details> + <summary> = Expandable/collapsible section (no JS needed!).
CODE
<header>
  <h1>My Blog</h1>
  <nav>
    <a href="#">Home</a>
    <a href="#">About</a>
  </nav>
</header>

<main>

  <section>
    <h2>Latest Posts</h2>

    <article>
      <h3>Post Title</h3>
      <p>Article body text goes here.</p>
    </article>

  </section>

  <aside>
    <h3>About the Author</h3>
    <p>Jane Doe, web developer.</p>
  </aside>

</main>

<!-- Figure with caption -->
<figure>
  <img src="https://picsum.photos/200/80"
       alt="Sample">
  <figcaption>Fig 1: A sample image</figcaption>
</figure>

<!-- Collapsible (no JS needed!) -->
<details>
  <summary>Click to expand FAQ</summary>
  <p>Answer text revealed here!</p>
</details>

<footer>
  <p>© 2024 My Blog</p>
</footer>
OUTPUT
💡 Why semantic tags? Using <header> instead of <div id="header"> tells Google and screen readers "this is the header". It makes your code readable, accessible, and better for SEO — all at once.
Chapter 11

Div, Span, ID & Class

<div> and <span> are generic containers with no visual meaning. They are used as hooks for CSS styling and JavaScript. id and class are how you target them.

📦

Block vs Inline Containers

<div> = Block element — takes the full width, starts on a new line. Used to group large sections.
<span> = Inline element — only as wide as its content, stays in line with text. Used to style specific words.
id="name" = Unique identifier. Only ONE element per page can have a given id. CSS: #header. JS: getElementById("header").
class="name" = Reusable label. MANY elements can share a class. CSS: .card.
CODE
<style>
  /* Target by id (unique, one per page) */
  #hero { background:#2c3e50; color:white; padding:20px; }

  /* Target by class (many elements) */
  .card { border:1px solid #ccc; padding:12px; margin:8px; }

  /* Span: style specific inline text */
  .highlight { background:yellow; font-weight:bold; }
</style>

<!-- div with id (unique) -->
<div id="hero">
  <h2>Hero Section</h2>
  <p>Welcome to my site.</p>
</div>

<!-- Multiple divs sharing same class -->
<div class="card">Card One content</div>
<div class="card">Card Two content</div>

<!-- Multiple classes on one element -->
<div class="card highlight">
  Card with TWO classes
</div>

<!-- span: styles words inside a paragraph -->
<p>The sky is
  <span class="highlight">bright blue</span>
today.</p>
OUTPUT
Chapter 12

Audio & Video

HTML5 introduced native audio and video support. No plugins needed — the browser plays media directly.

🎬

Audio & Video Tags with Controls

<audio> = Embeds an audio player. Attributes: controls (show play/pause), autoplay (play on load), loop (repeat), muted.
<video> = Embeds a video player. Same attributes as audio + width, height, poster (thumbnail image).
<source> = Specify multiple file formats for browser compatibility (some support MP4, others WebM, etc.).
The text inside (between tags) is shown only in browsers that don't support the element — fallback text.
CODE
<!-- AUDIO PLAYER -->
<audio controls>
  <!-- source: specify the audio file + format -->
  <source src="audio/song.mp3" type="audio/mpeg">
  <source src="audio/song.ogg" type="audio/ogg">
  <!-- Fallback text if browser can't play audio -->
  Your browser does not support audio.
</audio>

<!-- Audio with extra options -->
<audio controls loop muted>
  <source src="audio/background.mp3" type="audio/mpeg">
</audio>


<!-- VIDEO PLAYER -->
<video
  width="400"
  controls
  poster="images/thumbnail.jpg">
  <!-- Multiple formats for browser compatibility -->
  <source src="video/clip.mp4" type="video/mp4">
  <source src="video/clip.webm" type="video/webm">
  Your browser does not support video.
</video>

<!-- Video: autoplay + muted (required by browsers) -->
<video width="400" autoplay muted loop controls>
  <source src="video/demo.mp4" type="video/mp4">
</video>
⚠️ autoplay is blocked in most browsers unless muted is also set. This is to stop annoying auto-playing audio on websites.
💡 YouTube Embed: To embed a YouTube video, use an <iframe> (see Chapter 13) with YouTube's embed URL — not the <video> tag.
Chapter 13

Iframes — Embed Other Pages

An <iframe> (inline frame) embeds another webpage inside your page. Used to embed YouTube videos, Google Maps, social media widgets, and more.

🌐

Iframe Examples

src = The URL of the page to embed.
width / height = Size of the frame in pixels or %.
title = Accessibility description (required!).
allowfullscreen = Lets YouTube/Maps go fullscreen.
frameborder="0" = Removes the default border around the frame.
CODE
<!-- Embed a YouTube video -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="400"
  height="225"
  title="My YouTube Video"
  allowfullscreen
  frameborder="0">
</iframe>

<!-- Embed a Google Map -->
<iframe
  src="https://www.google.com/maps/embed?pb=YOUR_MAP_KEY"
  width="400"
  height="250"
  title="Location Map"
  frameborder="0">
</iframe>

<!-- Embed any website (if they allow it) -->
<iframe
  src="https://example.com"
  width="100%"
  height="300"
  title="Embedded Site">
</iframe>
OUTPUT (example.com embedded)
⚠️ Security: Many websites block embedding via X-Frame-Options headers. Google, Facebook, and most major sites refuse to be embedded. YouTube and Google Maps provide special embed URLs specifically for iframes.
Chapter 14

HTML Entities & Special Characters

Some characters have special meaning in HTML (like < and >). To display them as text, you use entity codes. Also used for symbols like ©, ₹, ♥.

🔣

Common HTML Entities

HTML entities start with & and end with ;
&lt; = < (less-than) | &gt; = > (greater-than) — used to show HTML code as text
&amp; = & (ampersand) | &nbsp; = non-breaking space (forces a space that won't wrap)
&copy; = © | &reg; = ® | &trade; = ™ | &euro; = € | &hearts; = ♥
CODE
<!-- Show < and > as visible text -->
<p>HTML tag looks like: &lt;p&gt;</p>

<!-- Copyright, trademark, registered -->
<p>&copy; 2024 My Company</p>
<p>Brand&reg; &nbsp; Brand&trade;</p>

<!-- Currency symbols -->
<p>Price: &euro;99 or &pound;85</p>

<!-- Fun symbols -->
<p>I &hearts; HTML &star; Web Dev</p>

<!-- Non-breaking space: words stay together -->
<p>Hello&nbsp;&nbsp;&nbsp;World</p>

<!-- Maths -->
<p>5 &lt; 10 &amp; 10 &gt; 5</p>
<p>&frac12; &frac14; &frac34;</p>
OUTPUT

Chapter — Bold / Strong Tag

Bold & Strong Tag

Both <b> and <strong> display text in bold — but they have different purposes. One is purely visual; the other carries meaning.

B

<b> vs <strong> — What is the difference?

<b> = Bold — visual styling only. Makes text appear bold. No extra meaning for search engines or screen readers.

<strong> = Bold + semantic importance. Tells the browser "this content is important." Screen readers may announce it differently. Google gives it slight SEO weight.

Simple rule: If the content is genuinely important — use <strong>. If you just want it to look bold with no special meaning — use <b>.
CODE
<!-- b: bold (visual only) -->
<p>This is <b>bold</b> text using <b>.</p>

<!-- strong: bold + important meaning -->
<p>This is 
  <strong>important</strong>
 text using <strong>.</p>

<!-- Both look the same visually -->
<p>
  <b>Visual bold</b>
   looks same as 
  <strong>Semantic bold</strong>
</p>

<!-- Real use case -->
<p>Warning:
  <strong>Do not close this window!</strong>
</p>

<!-- b with colour using style -->
<p>
  <b style="color: crimson;">
    Bold + red colour
  </b>
</p>

<!-- Nesting bold inside paragraph -->
<p>Price:
  <strong style="color: green;">₹499</strong>
   only!
</p>
LIVE OUTPUT
💡 Prefer <strong> over <b> in most situations — it carries semantic meaning, helps screen readers, and is better for SEO.

Chapter — target Attribute in Link Tag

target Attribute in <a> Link Tag

The target attribute on a link tag controls where the linked page opens — same tab, new tab, or a frame.

🎯

All target Values with Examples

target="_self" = Opens in the same tab (this is the default if you leave target out).
target="_blank" = Opens in a new browser tab — user stays on your site too.

   target="_blank"

  GitHub (safe new tab)
</a>

     ✅ External websites
     ✅ PDF downloads
     ✅ Docs user wants to keep open
     ❌ Do NOT use for internal page links -->

<!-- Internal link: no target needed -->

  About (internal — same tab)

Chapter — class and id in HTML

class and id in HTML

id gives a unique name to one single element. class gives a reusable name to many elements. Both are used to target elements with CSS styling and JavaScript.

🏷

id vs class — The Key Differences

id → Unique. Only one element on the page can have a given id value.
CSS selector: #header  |  JS: document.getElementById("header")
Use for: page sections, anchor jump links (#contact), unique elements like #logo, #main-title.

class → Reusable. Many elements can share the same class name.
CSS selector: .card  |  JS: document.getElementsByClassName("card")
Use for: repeated UI patterns — cards, buttons, menu items, badges.

Multiple classes: One element can have several classes separated by spaces: class="card featured large"
CODE
<style>
  /* id selector: # — unique, one per page */
  #page-title {
    color: navy;
    font-size: 26px;
  }

  /* class selector: . — reusable */
  .card {
    border: 1px solid #ccc;
    padding: 12px;
    margin: 6px;
    border-radius: 6px;
  }

  /* Second class for extra style */
  .featured {
    background: lightyellow;
    border-color: orange;
  }
</style>

<!-- id: unique element -->
<h1 id="page-title">Page Title</h1>

<!-- class: same class on many elements -->
<div class="card">Card One</div>
<div class="card">Card Two</div>

<!-- Two classes on one element -->
<div class="card featured">
  Card Three (featured!)
</div>

<!-- id used as anchor link target -->
<a href="#section2">
  Jump to Section 2
</a>
<!-- ... lots of content ... -->
<h2 id="section2">Section 2</h2>
LIVE OUTPUT
💡 Rule: Use id for unique elements (only once per page). Use class for patterns that repeat. Never use the same id value on two different elements — it will cause bugs in CSS and JavaScript.

Chapter — Favicon

Favicon — The Browser Tab Icon

A favicon is the tiny icon you see on the browser tab next to the page title. It also appears in bookmarks and browser history. Set it inside the <head> using a <link> tag.

⭐

How to Add a Favicon

A favicon is just a small square image — typically 32×32 pixels or 16×16 pixels.

File formats:
.ico = Old standard format. Works in all browsers including old IE.
.png = Modern choice. Clean and simple. 32×32 recommended.
.svg = Vector format — scales perfectly at any size (modern browsers only).

Where to put the favicon file: Place it in the same folder as your index.html (the root of your project).

Free favicon tools: favicon.io (from text, image, or emoji), realfavicongenerator.net
CODE — Add to <head>
<head>
  <meta charset="UTF-8">
  <title>My Website</title>

  <!-- Option 1: .ico file (classic) -->
  <link
    rel="icon"
    type="image/x-icon"
    href="favicon.ico">

  <!-- Option 2: .png file (modern, preferred) -->
  <link
    rel="icon"
    type="image/png"
    href="images/favicon.png"
    sizes="32x32">

  <!-- Option 3: .svg file (scales perfectly) -->
  <link
    rel="icon"
    type="image/svg+xml"
    href="images/favicon.svg">

  <!-- Best practice: provide all 3 -->
  <link rel="icon" type="image/svg+xml"
        href="favicon.svg">
  <link rel="icon" type="image/png"
        href="favicon.png" sizes="32x32">
  <link rel="icon" type="image/x-icon"
        href="favicon.ico">

</head>
HOW IT LOOKS
💡 Quick tip: The simplest favicon is a square image saved as favicon.png at 32×32 pixels, placed in the same folder as index.html, and linked with: <link rel="icon" href="favicon.png">
